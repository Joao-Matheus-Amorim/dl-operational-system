Fluxo de operação

Como o sistema trabalha para o gestor.

A automação precisa ser simples de entender: detectar, avisar, recomendar e executar somente quando estiver seguro.

01

Coleta

Busca métricas da Meta Ads e/ou dados da planilha, por cliente e período.

02

Análise

Calcula CPL, CTR, ROI, gasto, leads e identifica pontos críticos.

03

Alerta

Envia aviso no painel e no WhatsApp quando saldo ou criativo exige atenção.

04

Decisão

Gestor escolhe: pausar, manter ou criar campanha teste isolada para o criativo.

Implantação

Como sair do mock para operação real.

Como ainda não temos as APIs reais, a proposta respeita uma implantação por fases: primeiro validamos o produto, depois plugamos credenciais.

Fase 1 · Demo comercial

  • Dashboard responsivo rodando.
  • Dados demo por cliente.
  • Alertas mock no painel.
  • WhatsApp mock para apresentação.
  • Planos e escopo definidos.

Fase 2 · Integração real

  • Token Meta Ads.
  • IDs das contas de anúncio.
  • Service Account Google Sheets.
  • Z-API/WhatsApp.
  • Regras reais por cliente.

Fase 3 · Operação assistida

  • Alertas reais.
  • Relatórios semanais.
  • Análise de criativos.
  • Recomendações de ação.
  • Logs de cada decisão.

Fase 4 · Automação avançada

  • Pausa automática com regras.
  • Campanha isolada para criativo vencedor.
  • Webhook WhatsApp.
  • Banco histórico.
  • Dashboard com login.
Importante: neste momento, tudo que depende de API real fica para o final. O cliente pode aprovar o produto vendo o fluxo mock funcionando: dashboard, alerta, WhatsApp simulado e decisão operacional.

O melhor caminho é vender o Plus.

O Básico é a porta de entrada. Mas o Plus é o ponto ideal: já entrega dashboard, alertas, diagnóstico de criativos e fluxo de decisão sem exigir automações arriscadas logo no primeiro mês.
